# Scripts Guide

This folder contains the gameplay and UI logic for the Godot prototype. The
short version: scenes are the objects you place in the editor, scripts are the
behavior attached to those objects.

If you are coming from C++, GDScript may look loose at first. It is smaller and
more dynamic, but most files here are written with type hints and small modules
so they can still feel predictable.

## Folder Map

| Folder | Purpose |
| --- | --- |
| `audio/` | Shared audio data and helpers. |
| `camera/` | Reusable camera rigs. |
| `combat/` | Shared combat components that are not player-specific. |
| `debug/` | Editor and prototype visualization helpers. |
| `effects/` | Small gameplay visual effects. |
| `gathering/` | World resource metadata, such as trees. |
| `interaction/` | Shared hover, selection, and target feedback. |
| `inventory/` | Item definitions, stacks, and player inventory state. |
| `player/` | Player prefab modules: input, movement, animation, combat, gathering. |
| `ui/` | HUD, inventory, nameplates, and other player-facing UI. |
| `visuals/` | Reserved for shared visual helpers that are not player-only. |

## GDScript For C++ Readers

### Script Shape

```gdscript
class_name PlayerMovementMotor
extends Node

@export var movement_speed: float = 5.0
var _has_destination := false

func set_destination(destination: Vector3) -> void:
	_has_destination = true
```

- `extends Node` is similar to inheriting from a base engine class.
- `class_name PlayerMovementMotor` registers a global script type in Godot.
- `@export` exposes a value in the Godot Inspector.
- `func name(args) -> ReturnType:` declares a function.
- Indentation matters. There are no `{}` braces.

### Variables And Types

```gdscript
var count := 0
var speed: float = 3.2
const MAX_STACK := 999
```

- `var count := 0` asks Godot to infer the type from the value.
- `var speed: float = 3.2` gives an explicit type.
- `const` is for values that should not change.
- `_private_name` is our convention for internal state. GDScript does not make
  it private automatically.

### Nodes, Scenes, And Resources

Godot is built around a scene tree.

- `Node` is the base object.
- `Node3D` has a 3D transform.
- `Control` is UI.
- `Resource` is reusable data.
- `.tscn` files are saved scenes.
- `.gd` files are scripts.
- `.tres` files are saved resources.

Common node lookup syntax:

```gdscript
@onready var movement_motor = $Movement
var target := get_node_or_null(target_path)
```

- `$Movement` means "get my child node named Movement".
- `@onready` waits until the node is inside the scene tree before assigning.
- `NodePath` is a serialized path that can be edited in the Inspector.

### Lifecycle Functions

Godot calls these for you:

```gdscript
func _ready() -> void:
	pass

func _process(delta: float) -> void:
	pass

func _physics_process(delta: float) -> void:
	pass
```

- `_ready()` runs after the node enters the scene tree.
- `_process(delta)` runs every rendered frame.
- `_physics_process(delta)` runs on the physics tick and is best for movement.
- `_unhandled_input(event)` receives input not consumed by UI.

### Signals

Signals are Godot's event system.

```gdscript
signal channel_completed(context: Dictionary)

channeling.channel_completed.connect(_on_channel_completed)
channel_completed.emit(context)
```

Think of them like typed event callbacks. A script emits a signal; another
script connects a function to it.

### Dictionaries, Arrays, And Match

```gdscript
var item := {"id": "timber_t1", "quantity": 5}
var slots := []

match relationship:
	Relationship.FRIENDLY:
		return friendly_color
	_:
		return neutral_color
```

- `Dictionary` is similar to a map.
- `Array` is similar to a vector/list.
- `match` is close to `switch`.
- `_` means default case.

### Preload And Load

```gdscript
const ItemStackScript := preload("res://scripts/inventory/item_stack.gd")
var stack: Resource = ItemStackScript.new()
```

- `preload()` loads a file at parse time.
- `load()` loads a file at runtime.
- Use `preload()` for scripts/scenes/resources this file always needs.

### Common Godot Types

- `Vector2`: 2D position or size.
- `Vector3`: 3D position, direction, or velocity.
- `Color`: red, green, blue, alpha values from `0.0` to `1.0`.
- `NodePath`: saved path to another node.
- `PackedScene`: a scene you can instantiate.
- `Callable`: function reference used for signals or deferred calls.

### UID Files

Files like `player_controller.gd.uid` are generated by Godot. Do not edit them
by hand. If a new script creates a `.uid` file, commit it with the script.

## How To Approach A Change

1. Start at the nearest folder README.
2. Open the scene that owns the behavior, usually under `scenes/`.
3. Open the script attached to that scene or child node.
4. Follow signals and exported `NodePath` values to neighboring modules.
5. Keep changes in the smallest module that owns the behavior.

When in doubt, `player/controllers/player_controller.gd` is the player traffic
director, but it should not become the place where every system lives.
