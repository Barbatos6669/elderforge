# Design Boundaries

Elderforge is an original project. It can learn from the sandbox MMO genre, but it must earn its own identity.

## Allowed Inspiration

- Broad genre patterns such as gathering, crafting, player markets, classless progression, territory conflict, and risk-based travel.
- Publicly discussed design concepts that are common across many games.
- Original mechanics that solve similar player needs in a distinct way.

## Not Allowed

- Copied art, audio, fonts, icons, animations, maps, UI screens, item names, ability names, faction names, lore, quests, or dialogue.
- Reverse-engineered proprietary protocols, clients, servers, data files, or tools.
- Screenshots or traced designs used as source assets.
- Contributions that require commercial game files to build, test, or understand.

## Practical Rule

If a contributor can point at a commercial game's screen, file, database, wiki page, or asset and say "this is where the contribution came from," it probably does not belong here. Start from the player problem instead and design an Elderforge-native answer.
