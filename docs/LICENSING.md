# Licensing Notes

The repository uses the MIT License for source code.

Assets need special care because games mix code, art, audio, writing, data, and third-party tools. Before adding an asset, document:

- Who created it.
- What license applies.
- Whether attribution is required.
- Whether commercial use and modification are allowed.
- Where the original source can be found.

## Recommended Asset Policy

Prefer original assets created for Elderforge or assets under permissive licenses that allow modification and redistribution. Good candidates include CC0, CC BY 4.0, and other open licenses that are compatible with public game development.

Do not add assets ripped, traced, extracted, or recreated from commercial games.

## File-Level Notices

When an asset has its own license or attribution requirements, place a nearby notice file or add it to a future repository-wide `NOTICE.md`.
