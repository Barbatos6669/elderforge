# Roadmap

This roadmap is intentionally practical. Elderforge should become playable through small prototypes before large content production begins.

## Milestone 0: Repository Foundation

- Establish project license, contribution rules, and design boundaries.
- Keep the Godot project openable from a fresh clone.
- Create public issue templates and pull request expectations.

## Milestone 1: Local Play Prototype

- Basic top-down character controller.
- Camera, input, and interaction targeting.
- Simple test map with gatherable resource nodes.
- Inventory data model.
- One crafting recipe from gathered resources.

## Milestone 2: Multiplayer Spike

- Server-authoritative movement prototype.
- Client prediction and reconciliation experiment.
- Basic entity replication.
- Connection, disconnect, and reconnect behavior.
- Automated test scene for simulated clients.

## Milestone 3: Economy Loop

- Gather resources.
- Refine resources.
- Craft equipment.
- Equip items that affect abilities or stats.
- Drop, trade, and persist items safely.

## Milestone 4: Combat Prototype

- Ability definition format.
- Cooldowns, targeting, hit validation, and damage events.
- PvE enemy prototype.
- PvP duel prototype.
- Death, loot, and recovery rules.

## Milestone 5: World Persistence

- Account identity.
- Character save data.
- Inventory persistence.
- World state persistence for resources and placed structures.
- Migration strategy for saved data.
